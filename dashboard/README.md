# AI Trace Explorer

Internal dashboard that visualizes [Langfuse](https://langfuse.com) exports for three Kameleoon AI features, in one app with three tabs:

| Tab | Agent | What it shows |
| --- | --- | --- |
| **AI Asset Trace** | `targeting_agent` | Segments & triggers (keymoments) generated from natural-language prompts — conditions, match types, clarifications. |
| **AI Goals** | `goals` (Aria) | Conversion-goal tracking snippets generated by inspecting page HTML. |
| **Kai** | LangGraph copilot | Experiment-result summaries and Kameleoon documentation Q&A. |

Each tab has an **Exchange** view (chat-style trace browser with filters) and a **Patterns** view (clusters, charts, condition/outcome breakdowns).

## Stack

Vite + React + TypeScript, fully static. No backend — the UI fetches compact JSON datasets at runtime.

## Deployment

The app is deployed at **https://junaid-kameleoon.github.io/Claude/dashboard/**.

`vite build` writes its output into `../public/dashboard/`, so the repo's existing
"publish `public/` as-is" GitHub Pages workflow picks it up with **no workflow changes**.
The Vite `base` is `/Claude/dashboard/`.

## Password gate

A client-side login screen (`src/auth/Gate.tsx`) gates the UI. **This is not real
security** — GitHub Pages is static, so the data ships in the bundle. It only deters
casual visitors.

- Default placeholder password: `kameleoon-ai`
- To change it, hash a new password and paste it into `PASSWORD_HASH`:
  ```bash
  node -e 'console.log(require("crypto").createHash("sha256").update("YOUR_PW").digest("hex"))'
  ```

## Data generation

Raw Langfuse exports (~45–50 MB each) are **never** committed or shipped to the browser.
`scripts/generate-trace-data.mjs` reads them, groups observations by `traceId`, picks the
root observation per feature, extracts the useful fields, computes summary/pattern data,
and writes compact JSON to `public/`:

- `public/trace-data.json` — AI Asset targeting
- `public/kai-data.json` — Kai
- `public/goals-data.json` — AI Goals
- `public/datasets-manifest.json` — index of all three

By default it reads the exports from `~/Downloads`. Override per feature:

```bash
TARGETING_EXPORT=/path/a.json KAI_EXPORT=/path/b.json GOALS_EXPORT=/path/c.json npm run generate:data
```

The generated compact JSONs **are** committed, so the app builds without the raw exports present.

## Commands

```bash
npm ci                 # install
npm run generate:data  # regenerate compact datasets from raw exports (needs the raw files)
npm run dev            # local dev server (http://localhost:5173/Claude/dashboard/)
npm run build          # type-check + build into ../public/dashboard/
npm run preview        # preview the production build
```
